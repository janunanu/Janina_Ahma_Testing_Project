# Positive Test examples are below write test process steps for each below test case

	Create issue

	Pre Condition	The test project exists.
	Process Step One	Go to the "New Issue" page using the link: ${ISSUE_NEW_URL}
	Step Two	Type the title: ${ISSUE_TITLE_1}
	Step Three	Type the description: ${ISSUE_DESCRIPTION_1}
	Step Four	Select the label: ${ISSUE_LABEL}
	Step Five	Click the large 'Create issue' button at the bottom of the page.
	Verification or Validation Steps With Expected Data	
	The task should be created successfully. 
	Check that you see a success message (like "The issue has been successfully created.") 
	and the page switches to show the newly created task.
	
 	Update name of issue

	Pre Condition	The task with the title '${ISSUE_TITLE_1}' must exist.
	Process Step One	Go to the detail page for the task titled '${ISSUE_TITLE_1}'.
	Step Two	Look for and click the 'Edit' button (it usually looks like a pencil icon).
	Step Three	Change the Title box to: ${ISSUE_TITLE_UPDATED}
	Step Four	Click the 'Save changes' button to confirm the update.
	Verification or Validation Steps With Expected Data	
	The task's title should immediately change. 
	Check that the new title '${ISSUE_TITLE_UPDATED}' is displayed at the top of the page.

	Create second issue with privious updated name

	Pre Condition	The test project exists.
	Process Step One	Go back to the "New Issue" page: ${ISSUE_NEW_URL}
	Step Two	Type a new title: ${ISSUE_TITLE_2}
	Step Three	Type a brief description (e.g., Description for second issue).
	Step Four	Click the 'Create issue' button.
	Verification or Validation Steps With Expected Data	
	The second task should be created without any problems. 
	This confirms that updating the first task did not cause errors for new tasks.

    Delete first issue

	Pre Condition	The task with the updated title '${ISSUE_TITLE_UPDATED}' must exist.
	Process Step One	Go to the detail page for the task titled '${ISSUE_TITLE_UPDATED}'.
	Step Two	Click the 'Edit' button (pencil icon).
	Step Three	Look for and click the 'Delete issue' option (it is often in a dangerous settings section).
	Step Four	Click 'Yes' or 'Confirm' in the warning box to confirm you want to delete it.
	Verification or Validation Steps With Expected Data	
	The task should be permanently removed. 
	The page should switch to the list of all tasks, and a success message should appear 
	(like "The issue was successfully deleted.").
